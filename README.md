# Major

`major` 是一个平台无关的本科专业信息研究与咨询 Skill，面向中国大陆学生和家庭的使用频率做了强化，同时保留中国、韩国、日本、英国、美国、澳大利亚六国本科专业体系的比较与咨询能力。

它的定位不是“专业百科”或“专业名称查询器”，而是一个本科专业信息研究、来源核查、跨国对齐和个性化推荐助手。它帮助用户理解一个专业到底学什么、本科阶段如何训练、适合什么学生、和相近专业有什么差异、在不同国家如何命名和培养、未来可能通向哪些职业或研究生方向，以及用户本人是否适合。

## 核心定位

`major` 主要解决以下问题：

- 用户想了解某个本科专业的学习内容、课程结构、训练方式和能力要求；
- 用户想比较多个专业，例如计算机科学、软件工程、数据科学、信息系统；
- 用户想比较同一类专业在中国、韩国、日本、英国、美国、澳大利亚的差异；
- 用户只有兴趣方向，例如 AI、心理学、商科、传媒、设计、医学，但不知道对应哪些本科专业；
- 用户想根据自己的年级、选科、成绩、兴趣、语言、预算、职业偏好和风险偏好获得本科专业推荐；
- 用户想核查具体大学项目、招生专业、专业组、选科要求、分省计划、课程设置、学费、截止日期、认证或就业数据；
- 用户想把专业信息整理成表格、报告、咨询建议或来源核查材料。

默认回答语言是中文，默认聚焦本科阶段。除非用户明确询问，`major` 不默认扩展到研究生申请、职业资格申请、签证移民办理或就业结果预测。

## 目标用户

`major` 适合以下用户：

- 中国大陆高中生；
- 国际课程学生，例如 IB、A-Level、AP、国际学校学生；
- 本科申请者和准大学生；
- 家长；
- 留学顾问；
- 升学规划师；
- 需要整理本科专业资料的教育机构；
- 想理解不同国家本科专业体系差异的用户。

其中，中国大陆学生是最主要的高频使用对象。因此，Skill 对中国大陆高校本科招生专业、招生目录、专业组、选科要求、分省计划、培养方案和第三方专业排名参考做了额外建模。

## 覆盖范围

`major` 覆盖六个国家或地区体系：

- 中国；
- 韩国；
- 日本；
- 英国；
- 美国；
- 澳大利亚。

核心教育阶段为本科。它可以在专业介绍中提及研究生方向，但不会把研究生申请作为默认主题。涉及医学、护理、药学、心理、法律、教育、建筑、工程、会计、兽医、社会工作等受监管路径时，它会提醒用户：本科专业学习不等于自动获得执业资格，具体资格路径必须核查官方监管机构、专业认证机构和目标大学。

## 能力模块

`major` 由多个模块共同构成，不是单纯提示词集合。

- `SKILL.md`: Codex/Agent Skill 的根入口，负责最小化运行时路由和渐进披露。
- `skill.yaml`: 平台无关元数据，记录名称、版本、国家范围、默认行为和时效规则。
- `instructions/`: 核心行为规则，包括咨询逻辑、推荐规则、检索规则、国家说明、安全边界和隐私最小化。
- `prompts/`: 可复用任务模板，例如专业介绍、专业比较、推荐、来源总结和国家研究。
- `schemas/`: 结构化输入输出协议，包括用户画像、专业画像、推荐结果、来源记录、claim 验证、变化记录和中国专业库记录。
- `data/`: 规则和种子数据，包括国家术语、专业分类、专业别名、官方来源、权重、认证规则、问题库和中国大陆专业库元数据。
- `workflows/`: 任务流程，包括单专业介绍、个性化推荐、多国家比较、来源核查、院校项目查询和变化检测。
- `algorithms/`: 平台无关算法说明，包括意图路由、工作流编排、时效检索、混合检索、来源评分、用户画像补全、适配评分、风险评分、约束过滤、偏好学习、推荐排序、专业相似度、知识图谱、跨国对齐、变化检测、报告生成和中国专业库处理。
- `scripts/`: 可执行辅助脚本，用于评分、ID 生成、记录验证、评测检查、Skill 结构验证和授权中国专业数据导入。
- `adapters/`: Claude、OpenAI Agents 和 MCP 的轻量适配文件。
- `evals/`: 机器和人工评测材料，包括官方兼容 `evals.json`、人类可读 YAML 测试集、rubric、fixtures 和 sample outputs。

## 典型工作流

### 单专业介绍

当用户问“介绍一下心理学专业”“计算机科学本科主要学什么”时，`major` 会：

1. 判断意图是否为 `explain_major`；
2. 识别国家、专业、是否涉及具体大学或认证；
3. 对稳定概念进行结构化解释；
4. 如果涉及当前大学项目、课程、招生、认证或就业数据，执行 current/next-cycle 检索；
5. 使用知识图谱整理课程、技能、职业、相近专业和资格路径；
6. 输出定义、本科课程、适合/不适合人群、相近专业差异、国家差异、未来方向、误区、来源和建议。

### 个性化推荐

当用户问“我喜欢 AI，适合什么专业”“我数学一般但想学商科”时，`major` 不会直接给最终结论。它会先判断用户画像是否足够。

用户画像包括：

- 目标国家；
- 当前年级和课程体系；
- 选科和学术背景；
- 擅长科目和薄弱科目；
- 兴趣方向；
- 喜欢和不喜欢的任务；
- 职业偏好；
- 语言能力；
- 预算水平；
- 风险偏好；
- 限制条件；
- 对就业、收入、稳定性、科研、移民、申请难度等因素的优先级。

初次反问最多 5 个问题。问题不是固定模板，而是通过 `active_profile_completion` 选择信息增益最大的字段。随后通过 `constraint_filtering`、`major_fit_scoring`、`risk_scoring`、`preference_learning` 和 `recommendation_ranking` 生成推荐分组。

推荐结果会分为：

- 核心推荐；
- 相邻备选；
- 谨慎考虑；
- 不太建议；
- 需要补充信息后判断。

每个推荐都应说明推荐理由、适合点、风险点、替代专业、适合关注的国家或体系、需要进一步验证的信息和下一步行动建议。

### 具体院校或招生信息核查

当用户问“某大学现在有哪些计算机相关专业”“某省物化生能不能报某专业”“今年专业组有什么变化”时，`major` 会优先进入来源核查或院校项目查询工作流。

对于中国大陆高校，它会特别关注：

- 省份或考试地区；
- 目标入学年份；
- 招生周期；
- 选科组合；
- 批次或申请路径；
- 专业组；
- 分省招生计划；
- 专业代码和院校专业组代码；
- 来源口径是全国专业目录、大学招生目录、省考试院计划、专业组、选科要求还是培养方案。

如果缺少会改变结论的 scope，`major` 不应给最终报考结论，而应先说明缺口并提出必要问题。

## 中国大陆大学专业数据存储库

`major` 包含 `data/china_major_repository/`，用于中国大陆本科专业高频查询的候选初筛。

该目录包含：

- `metadata.yaml`: 数据集元信息、授权状态、使用规则和局限；
- `sources.yaml`: 软科、教育部、阳光高考等来源类型和使用边界；
- `sample_records.yaml`: 合成样例数据，仅用于结构测试；
- `import_mapping.yaml`: 授权 CSV/JSON 导入字段映射；
- `README.md`: 中国专业库使用边界说明。

该专业库的设计原则是：

- 本地库只用于候选院校、候选专业和排名参考；
- 不内置完整软科 2025 排名明细；
- 不从网页复制未授权完整榜单；
- 部署方或用户可以导入有授权的 CSV/JSON；
- 软科 2025 中国大学专业排名等第三方排名只能作为专业实力参考；
- 第三方排名不等于录取难度、就业保证、当前招生、选科要求、专业组、分省计划或培养方案；
- 当前招生事实仍必须通过高校招生网、阳光高考、省级考试院、教育部、高校教务处或院系官网等官方 current/next-cycle 来源核验。

相关算法包括：

- `china_major_repository_lookup`: 从本地库生成候选院校和候选专业；
- `china_major_repository_import`: 导入授权 CSV/JSON；
- `china_major_repository_validation`: 校验记录、授权、来源 ID 和使用边界；
- `china_major_repository_ranking_reference`: 管理软科等第三方排名参考的边界；
- `china_admissions_scope_resolver`: 解析中国大陆招生 scope。

## 来源与时效原则

`major` 对具体事实采用“先检索，再总结”的原则。

当问题涉及以下内容时，必须优先使用 current/next-cycle 官方来源：

- 具体大学项目；
- 当前招生专业；
- 招生章程和招生简章；
- 招生专业目录；
- 分省招生计划；
- 专业组；
- 选科要求；
- 培养方案；
- 课程设置；
- 学费；
- 申请截止日期；
- 专业认证；
- 职业资格路径；
- 就业数据；
- 签证、工签、移民或政策变化。

`2025-01-01` 是最低时效底线，不等于“当前”。如果当前日期已经进入 2026 或以后，而只找到 2025 来源，必须判断它是否仍适用于当前或下一招生周期；不能把旧的 2025 页面默认包装成最新信息。

来源优先级大致为：

1. 大学官网、本科招生网、教务处、官方课程目录；
2. 院系官网；
3. 教育部、政府教育部门、省级考试院、阳光高考、官方申请平台；
4. 专业认证机构；
5. 职业统计机构；
6. 行业协会；
7. 可信教育平台或媒体；
8. 论坛、博客、无来源内容。

论坛、博客、聚合平台、营销页面和未标注来源的数据不能作为当前事实的唯一依据。

## Claim 验证与不确定性

`major` 会把关键结论拆成 claim，并判断每个 claim 的状态：

- `supported`: 有可靠来源直接支持；
- `partially_supported`: 有来源支持部分结论；
- `conflicting`: 来源之间冲突；
- `not_found`: 未找到支持来源；
- `outdated`: 来源过期；
- `uncertain`: 来源不足或适用范围不清。

确定性分为：

- `high`: current/next-cycle 官方来源直接支持；
- `medium`: 来源可靠但缺少一个关键字段，例如明确年份、更新时间或交叉验证；
- `low`: 来源不够官方或只能间接支持；
- `unknown`: 没有可靠来源支持。

`low` 只能作为参考，`unknown` 必须说明无法确认，`outdated` 和 `not_found` 不得作为当前结论输出。

## 专业推荐逻辑

`major` 的推荐不是简单按热门、排名或就业传闻排序。推荐会综合：

- 兴趣匹配；
- 能力匹配；
- 学习过程匹配；
- 职业目标匹配；
- 国家体系匹配；
- 约束条件匹配；
- 风险评分；
- 用户偏好更新；
- 来源确定性；
- 专业多样性。

内部适配度评分参考：

```text
fit_score =
0.30 * interest_match
+ 0.20 * ability_match
+ 0.15 * study_process_match
+ 0.15 * career_goal_match
+ 0.10 * country_system_match
+ 0.10 * constraint_match
```

风险评分参考：

```text
risk_score =
0.25 * skill_gap_risk
+ 0.20 * study_pressure_risk
+ 0.20 * admission_risk
+ 0.15 * employment_uncertainty_risk
+ 0.10 * qualification_risk
+ 0.10 * budget_or_language_risk
```

评分只是辅助解释，不是绝对结论。`major` 不会承诺录取、就业、薪资、签证、移民或职业资格结果。

## 跨国专业对齐

不同国家对“专业”的组织方式不同，`major` 不会简单把翻译相同的名称视为等价。

典型术语差异包括：

- 中国：本科专业、学科门类、专业类、专业目录、招生专业、培养方案；
- 韩国：학과、전공、학부、단과대학、모집단위；
- 日本：学部、学科、専攻、コース、課程；
- 英国：course、programme、subject、degree、module；
- 美国：major、minor、concentration、department、undergraduate program；
- 澳大利亚：course、degree、major、stream、specialisation。

跨国比较时，`major` 会判断课程结构、学制、核心课程、申请单位、专业层级、作品集/实验/临床/实习要求、职业路径、认证要求和资格路径，而不是只比较中文译名。

## 变化检测

`major` 支持比较不同年份的专业或招生信息变化，尤其适合中国大陆高校每年变化较明显的场景。

可检测的变化包括：

- 新增专业；
- 停招或移除；
- 专业更名；
- 专业代码变化；
- 归属学院变化；
- 大类招生变化；
- 专业组变化；
- 分省招生计划变化；
- 选科要求变化；
- 培养方案变化；
- 学费变化；
- 中外合作项目变化；
- 实验班、拔尖班、强基、专项计划变化；
- 课程模块变化；
- 认证状态变化。

变化检测要求 old source set 和 new source set 口径一致。不得把全国专业目录、省考试院分省计划、高校招生目录、专业组和培养方案混为同一个变化结论。

## 结构化记录与 ID

为了支持可追踪回答，`major` 使用稳定 ID 连接来源、claim、知识图谱和变化记录。

- `source_id`: 来源记录唯一标识；
- `claim_id`: 声明验证记录唯一标识；
- `node_id`: 知识图谱节点标识；
- `edge_id`: 知识图谱边标识；
- `change_id`: 年份变化记录标识；
- `record_id`: 中国专业库记录标识；
- `ranking_reference_id`: 中国第三方排名参考标识。

这些 ID 让回答可以追踪到：某个结论由哪个来源支持、来源是否当前、是否满足 2025 最低底线、是否只作为参考、是否存在冲突或局限。

## 脚本

`scripts/` 提供确定性辅助能力。

- `algorithm_utils.py`: 来源评分、问题选择评分、专业适配评分、风险评分、确定性分类、相似度分类和变化 diff；
- `china_major_repository_import.py`: 授权中国专业数据导入和数据集验证；
- `validate_records.py`: 校验 source、claim、graph、change 等记录的 Schema 和 ID 引用；
- `eval_runner.py`: 校验 eval YAML/JSON 同步、正负断言分离和 sample output 行为约束；
- `validate_skill.py`: 校验整个 Skill 的结构、Schema、adapter 资源、算法注册和行为 fixture；
- `test_algorithm_utils.py`: 单元测试。

常用验证命令：

```bash
python3 major/scripts/validate_skill.py
python3 -m unittest discover major/scripts -p 'test_*.py'
python3 major/scripts/eval_runner.py --mode all
python3 /Users/drew/.codex/skills/.system/skill-creator/scripts/quick_validate.py major
```

授权中国专业数据导入示例：

```bash
python3 major/scripts/china_major_repository_import.py \
  --input authorized_majors.csv \
  --output china_major_repository_dataset.json \
  --authorization-status authorized \
  --data-year 2025
```

## 评测体系

`evals/` 包含机器和人工评测材料。

- `evals/evals.json`: 官方风格评测入口，包含 `skill_name` 和 `evals`；
- `evals/test_cases.yaml`: 人类可读测试用例；
- `evals/rubric.md`: 评分标准；
- `evals/fixtures/`: ID 追踪和记录验证样例；
- `evals/sample_outputs/`: 行为 lint 样例。

评测覆盖：

- 单专业介绍；
- 相近专业比较；
- 兴趣词扩展；
- 个性化推荐；
- 中国大陆招生目录、选科、专业组和变化检测；
- 英美澳日韩专业体系；
- 找不到 current/next-cycle 来源时的 fallback；
- 来源冲突；
- claim 验证；
- 不确定性等级；
- 中国专业库候选和软科排名参考边界；
- 隐私最小化和最多 5 个反问。

## 安全边界

`major` 必须遵守以下边界：

- 不保证录取；
- 不保证就业；
- 不保证薪资；
- 不保证签证、工签或移民结果；
- 不保证职业资格；
- 不把本科专业等同于执业资格；
- 不把第三方排名等同于录取难度或就业结果；
- 不把旧来源包装成当前信息；
- 不要求用户提供身份证号、详细家庭资产、精确住址等无关敏感信息；
- 用户拒绝补充画像时，继续给有限建议并明确局限。

## 平台迁移

核心能力只依赖 Markdown、YAML、JSON 和少量 Python 脚本。平台适配文件只负责迁移，不承载核心业务逻辑。

迁移到新平台时，优先保留：

1. `instructions/`
2. `prompts/`
3. `schemas/`
4. `data/`
5. `workflows/`
6. `algorithms/`
7. `scripts/`

再按平台要求补充 adapter。

现有 adapter：

## 维护原则

- 根 `SKILL.md` 保持精简，只做入口和条件加载路由。
- 详细规则放在 `instructions/`、`algorithms/`、`workflows/` 和 `data/`。
- 新增算法必须注册到 `data/algorithm_registry.yaml`。
- 新增 Schema 必须是合法 JSON Schema，并纳入 adapter 和验证脚本。
- 新增数据源必须说明来源类型、时效、授权状态、使用边界和局限。
- 涉及中国大陆招生事实时，必须区分全国目录、省考试院、高校招生目录、分省计划、专业组、选科要求和培养方案。
- 涉及第三方排名时，必须明确它只是参考，不是官方招生事实。
